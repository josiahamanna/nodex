## Facade imports (`@nodex/ui` / `@nodex/date`)

Imports are **only** allowed from `@nodex/ui` or `@nodex/date`; they are stripped at compile time and components come from the host map.

<Button>Primary button</Button>

<Button variant="outline">Outline button</Button>

<NodexFormatIso value="2024-06-01T12:00:00.000Z" />

## Internal note links

Use hash routes so the shell can intercept navigation (adjust IDs/paths):

- [Note by id](#/n/YOUR_NOTE_ID)
- [Note + heading slug](#/n/YOUR_NOTE_ID/some-heading)
- [VFS path](#/w/path/to/note)

## Same-page heading link

Jump targets use generated heading ids. After you add a heading below, link to it with a hash that matches its slug (e.g. `#mdx-limitations`).

### MDX limitations (workspace notes)

**Not allowed** in workspace MDX: `{expressions}`, `export`, dynamic `import()`, side-effect-only imports, and arbitrary module specifiers. Raw `<script>` / `<iframe>` in MDX are blocked or replaced by the host.
