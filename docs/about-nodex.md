# About Nodex

Nodex is a notebook-native studio for technical work: notes, markdown docs, and interactive notebooks—designed for people who think in code.

## What makes it different

### Keyboard-first, command-driven

Nodex is built around a simple idea: **the UI is a surface for commands**.

- Navigate and act primarily through **commands and keyboard shortcuts**
- Treat panels, views, and workflows like **composable actions** instead of rigid screens
- Stay in flow: fewer context switches, more direct control

### Programmer-first note taking

Nodex is optimized for builders:

- **Structure your knowledge like a product**, not a folder
- Keep decisions, experiments, and implementation notes close to the work
- Use a workspace that feels more like an IDE—**but for knowledge**

### Extensible by design

Nodex is meant to evolve with you:

- Extend and enhance features through **plugins and contributions**
- Add new workflows by introducing **new commands**
- Treat the app as a toolkit you can shape over time

### Self-documenting, in-app docs

Documentation isn’t an afterthought. Nodex keeps guides and command/API documentation **inside the app**, so help is available where you’re already working.

## A useful mental model

If you like tools that reward learning their keys and composing small actions into powerful workflows, Nodex will feel familiar.

**Think of it as “Emacs for note-taking.”**
